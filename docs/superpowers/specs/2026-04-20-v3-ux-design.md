# Canvas Annotator V3 — UX 交互重设计

**日期：** 2026-04-20  
**背景：** V2 发现五个体验问题（详见下方），本次重设计统一解决。

---

## 问题清单（来自实测）

1. 摘录后需手动点同步才入 Canvas，有摩擦
2. 状态栏静默开关太隐蔽（右下角）
3. Canvas 节点点击没有跳回 md（`is-focused` class 名 bug）
4. 取消摘录需手动逐字删除
5. 点击摘录的高亮跳转，把当前 tab 替换掉了

---

## 设计决策

### 1. 浮动 toolbar（替代命令 + 状态栏静默开关）

**触发：** mouseup 后，若有选区 → 在选区上方弹出浮动 div。

**按钮逻辑：**
- 普通文字选中 → 显示 `[✎ 摘录]`
- 光标/选区在已有 `<mark>` 内 → 显示 `[✂ 取消]`（剥掉 `<mark>` 标签，保留纯文本）

**定位：** `window.getSelection().getRangeAt(0).getBoundingClientRect()` 拿到选区坐标 → `position: fixed` 绝对定位 div 跟随。滚动时隐藏（监听 `scroll`）。

**沉浸模式：** 插件设置项 `autoAnnotate`（默认关）。开启后 mouseup 有选区直接摘录，不弹 toolbar。

**移除：** 状态栏 `✎ OFF/摘录` 开关（V2 引入，现在由 toolbar 和 `autoAnnotate` 设置替代）。

---

### 2. 顶部固定 panel（CM6 showPanel）

编辑器内容区顶部常驻一个条：

```
[ ⟳ 同步到 Canvas ]
```

- 点击 → 触发 `syncAnnotations()`
- 仅在 Markdown 编辑器中显示（`showPanel` facet 自动限制作用域）
- Ribbon 图标保留作为备用入口

---

### 3. 两个新设置项

| 设置 key | 类型 | 默认 | 说明 |
|----------|------|------|------|
| `autoAnnotate` | boolean | false | 沉浸模式：mouseup 有选区直接摘录，不弹 toolbar |
| `autoSync` | boolean | false | 摘录后若检测到有打开的 Canvas → 立即同步（无需手动点同步）|

---

### 4. Bug 修复

#### 4a. 跳转占用当前 tab

**现状：** `getLeaf(false)` 在当前 leaf 打开文件，把 md 内容替换掉了。

**修复：**
```typescript
// md → canvas 跳转
const existingLeaf = this.app.workspace.getLeavesOfType("canvas")
  .find(l => (l.view as any)?.file?.path === file.path);
const leaf = existingLeaf ?? this.app.workspace.getLeaf("split");
await leaf.openFile(file);

// canvas → md 跳转（同理）
const existingMdLeaf = this.app.workspace.getLeavesOfType("markdown")
  .find(l => (l.view as any)?.file?.path === file.path);
const mdLeaf = existingMdLeaf ?? this.app.workspace.getLeaf("split");
await mdLeaf.openFile(file);
```

逻辑：先找已打开的对应 leaf → 有就复用 → 没有才 split 新建。

#### 4b. Canvas → md 跳回失效

**现状：** `n.nodeEl?.hasClass?.("is-focused")` 可能不是正确的选中节点 class。

**修复方向：** 调研 Canvas 内部实际使用的 selected/focused class 名（通过 DevTools 实测），改为正确 class 或改用 `canvas.selection` API（如果存在）。需要在 Obsidian 中实测确认，故列为待调研项。

## 验证标准

1. `npx vitest run` — 全部通过
2. `npm run build` — 构建成功
3. Obsidian 手动测试：
   - 选中文字 → 浮出 toolbar → 点摘录 → 高亮出现
   - 光标在高亮内 → 浮出 toolbar → 点取消 → 高亮变纯文本
   - `autoAnnotate` 开启 → 选中松开即摘，不弹 toolbar
   - `autoSync` 开启 + 有 Canvas 打开 → 摘录后 Canvas 立即出现节点
   - 顶部 panel 出现 → 点同步按钮 → 节点创建
   - md 点击高亮 → 跳到 Canvas 已有 leaf（不替换当前 tab）
   - Canvas 选中节点 → 跳转命令 → 跳到 md 对应位置
