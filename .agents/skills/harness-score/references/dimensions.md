# Harness Score Dimensions

## Entry Clarity

Checks whether `AGENTS.md` is short, actionable, project-specific, and uses one-layer indexing.

## Workflow Integrity

Checks whether all tasks share one main workflow and whether PGE or topic-specific flows are subflows rather than bypasses.

## Gate Coverage

Checks Coding Start, context, size/risk, dirty worktree, verification, circuit breaker, commit, and git gates.

## Rule Ownership

Checks whether every rule has one owner file and whether files avoid duplicate truth sources.

## Project Specificity

Checks whether commands, stack, structure, risks, APIs, DB, and deployment guidance match the actual project.

## Evolvability

Checks instruction governance, failures, glossary, active/stub/layer metadata, and a clear route for project-grown rules.

## Verification & Review

Checks testing guidance, review independence, `verify_cmd` recording, fallback policy, and regression expectations.

## Instruction Quality

Checks concise style, action-first wording, low redundancy, limited table use, and absence of vague slogans.
